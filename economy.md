# Основные экономические механики

BNB City использует дефляционную модель перераспределения: новые депозиты финансируют выплаты существующим игрокам, а система комиссий и эпох обеспечивает устойчивость.

## Адаптивная ставка (Adaptive Rate)

Ключевая формула доходности:

```
adaptiveRate = baseRate x (poolBalance / activeDeposits)
```

- **baseRate** -- базовая ставка текущей эпохи (от 2% до 5%)
- **poolBalance** -- текущий баланс пула контракта
- **activeDeposits** -- сумма всех активных депозитов

Ставка зажата в диапазон **0.5% -- 10%** в день. Когда пул здоров (баланс высок относительно депозитов) -- ставка выше. Когда пул под нагрузкой -- ставка снижается. Это саморегулирующийся механизм.

## Эффективная ставка по тирам

Каждый тир здания имеет собственный модификатор ставки:

```
effectiveRate = adaptiveRate x TIER_RATE_MOD[tier] / 100
```

| Тир | Rate Mod | Пример при adaptiveRate = 4% |
|-----|----------|------------------------------|
| T0 | x1.20 | 4.80% |
| T1 | x1.15 | 4.60% |
| T2 | x1.10 | 4.40% |
| T3 | x1.05 | 4.20% |
| T4 | x1.00 | 4.00% |
| T5 | x0.85 | 3.40% |
| T6 | x0.70 | 2.80% |
| T7 | x0.55 | 2.20% |

Младшие тиры зарабатывают процентно больше, старшие -- абсолютно больше.

## Cap (потолок выплат)

Каждое здание имеет максимальную сумму выплат:

```
maxPayout = deposit x epochCap x TIER_CAP_MOD[tier] / 100
```

Значения Cap Mod по тирам:

| Тир | Cap Mod |
|-----|---------|
| T0 | x1.25 |
| T1 | x1.20 |
| T2 | x1.18 |
| T3 | x1.15 |
| T4 | x1.05 |
| T5 | x1.00 |
| T6 | x0.80 |
| T7 | x0.80 |

### Cap Bonus

Каждый compound добавляет **+1% к капу**. Максимальный бонус: **+50%**.

```
maxPayout = deposit x epochCap x TIER_CAP_MOD / 100 x (100 + capBonus) / 100
```

Пример: T0 здание в эпохе E0, после 30 компаундов:
- deposit x 3.0 x 1.25 x 1.30 = deposit x 4.875

## 12-часовой цикл

Доход накапливается **12 часов**, затем останавливается до действия игрока (compound или claim). Расчёт ленивый: доход вычисляется в момент действия, а не тикает каждый блок.

## Продуктивность

Влияет на скорость заработка:

| Параметр | Значение |
|----------|----------|
| Стартовая | 100% |
| Максимальная | 500% (через overcharge) |
| Compound | +15% |
| Claim | -15% (масштабируется) |
| Пассивное восстановление | +10% в день |

Продуктивность ниже 100% замедляет заработок пропорционально. Выше 100% -- ускоряет.

## Комиссии

| Операция | Комиссия |
|----------|----------|
| Депозит (постройка) | 10% |
| Claim (снятие) | 13--16% (10% dev + 3--6% pool) |
| Вывод реф-баланса | 20% |
| P2P маркетплейс | 10% |
| Sellback | 15--100% (масштабируется) |
