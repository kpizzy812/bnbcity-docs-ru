# Смарт-контракт и безопасность

## Архитектура

BNB City работает на **одном неизменяемом смарт-контракте** на BNB Smart Chain:

- **Один контракт** -- вся логика игры в одном месте (~1170 строк Solidity)
- **Без прокси** -- нет возможности обновления, админ не может менять правила
- **Без паузы** -- контракт работает вечно, не может быть остановлен
- **Без внешних зависимостей** -- без наследования, без импортов (кроме ReentrancyGuard)
- **Solidity 0.8.24** -- встроенная защита от переполнения
- **viaIR: true**, optimizer 200 runs

## Что может админ

У админа (мэра) ограниченные привилегии:
- `setDevWallet()` -- сменить адрес получения dev-комиссий

Админ **не может**:
- Менять комиссии, ставки, cap'ы или пороги эпох
- Ставить контракт на паузу
- Выводить средства игроков
- Изменять параметры зданий после создания
- Менять реферальные проценты
- Менять стоимость синков или земли

## Все параметры захардкожены

Ключевые экономические параметры зашиты в контракт как константы:

| Параметр | Значение | Константа |
|----------|----------|-----------|
| Deposit fee | 10% | DEPOSIT_FEE_BPS = 1000 |
| Claim dev tax | 10% | CLAIM_TAX_DEV_BPS = 1000 |
| Реф-вывод налог | 20% | -- |
| P2P комиссия | 10% | -- |
| Тиры (8 штук) | Зафиксированы | TIER_DEPOSIT, TIER_CAP_MOD, TIER_RATE_MOD |
| Эпохи (7 штук) | Зафиксированы | EPOCH_THRESHOLD, EPOCH_BASE_RATE, EPOCH_CAP, EPOCH_POOL_TAX |
| Цены земли | Зафиксированы | LAND_PRICE |

Никто, включая создателя контракта, не может изменить эти значения после деплоя.

## Меры безопасности

| Вектор атаки | Защита |
|-------------|--------|
| Reentrancy | ReentrancyGuard + паттерн Checks-Effects-Interactions |
| Flash loan манипуляция | Ставка основана на `activeDeposits`, не на `balance` |
| Sandwich-атаки | Фиксированные суммы депозитов, нет slippage |
| Dust/spam-атаки | Минимум 0.001 BNB депозит + стоимость земли |
| Self-referral | Проверка `referrer != msg.sender` |
| Реферальные петли | Реферер фиксируется один раз, неизменяем |

## Гарантии on-chain

Эти свойства обеспечиваются кодом смарт-контракта:

1. **Суммарные выплаты не могут превысить баланс контракта** -- инвариант платёжеспособности
2. **Cap и tax здания фиксируются при создании** -- нет ретроактивных изменений
3. **Адаптивная ставка всегда между 0.5% и 10%** -- зажатый диапазон
4. **Total claimed каждого здания не может превысить его cap**
5. **Active deposits корректно уменьшаются** при завершении зданий
6. **Координаты земли проверяются on-chain** -- нет коллизий участков

## Тестирование

Контракт протестирован с помощью:
- **Hardhat тесты** (`test/BNBCity.test.ts`) -- интеграционные тесты с манипуляцией времени
- **Foundry fuzz тесты** (`test/foundry/BNBCity.fuzz.t.sol`) -- рандомизированное тестирование инвариантов (256+ запусков)
- **Testnet E2E** -- мульти-кошельковые сценарии на BSC Testnet
